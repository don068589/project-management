# Contributing to Project Management System

Thank you for your interest in contributing!

## How to Contribute

### Reporting Issues

- Use [GitHub Issues](https://github.com/openclaw/project-management/issues)
- Describe the problem clearly
- Suggest improvements if possible

### Submitting Changes

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test with real projects if possible
5. Submit a pull request

### Documentation Guidelines

When adding or modifying documentation:

1. Follow the existing structure in `docs/`
2. Keep it practical and actionable
3. Include examples where helpful
4. Update `data_structure.md` if adding new files

### Template Guidelines

When adding or modifying templates:

1. Place in `templates/` directory
2. Include clear instructions
3. Use placeholders for user-specific content
4. Keep it simple and focused

## Project Structure

```
docs/           → Detailed rules and guidelines
templates/      → Ready-to-use project documents
tools/          → Optional utilities
```

## Questions?
